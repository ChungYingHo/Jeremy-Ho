如果是對 markdown 還沒甚麼接觸經驗的建議先去下述網站補一下基礎：
1. [Markdown文件](https://markdown.tw/)
2. [Docusaurus 官方教學](https://docusaurus.io/zh-CN/docs/markdown-features)

另外 docusaurus 支援 MDX 格式，MDX 是 Markdown 與 JSX 的結合，意圖是在 Markdown 中使用 React 元件，我個人是滿愛用的，因為比 MD 可以多很多撰寫風格。  
使用 MDX 時推薦安裝 [MDX VScode 插件](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx)。

想讓文章不死板、添加人情味的話，[markdown 表情符號](https://gist.github.com/rxaviers/7360908)也有列出適用在 MD 中的表情符號編碼。

## markdown 縮放式區塊
<details>
<summary>Example</summary>

Here is an example
</details>
```markdown
<details>
<summary>Example</summary>

Here is an example
</details>
```
:::warning
記得 `<summary></summary>` 下面要空一行。
:::

## Code Block highlight
這裡的 highlight 指的不是 code block 自動對特定語言進行高亮，而是指撰寫者可以在 code block 中透過「註解」指定特定行數進行高亮顯示。

:::warning
這不是 markdown 自帶的功能，所以不要傻傻的在 github 或 gitlab 的 readme、MR 或 PR 上使用，是沒用的。
:::

1. 下一行 highlight 提示：`highlight-next-line`
```js title="highlight-next-line"
// highlight-next-line
console.log('This line will be highlighted')
console.log('This line will not be highlighted')
```

2. 指定段落 highlight：`highlight-start` 與 `highlight-end`
```js title="highlight-start / highlight-end"
// highlight-start
console.log('This line will be highlighted')
console.log('This line will also be highlighted')
// highlight-end
console.log('This line will not be highlighted')
```
